# CLAUDE.md

## Project

Personal portfolio and blog — static Next.js site deployed to GitHub Pages via `output: 'export'`.

## Commands

- `npm run dev` — dev server (also generates RSS)
- `npm run build` — production build to `out/` (also generates RSS)
- `npm run lint` — ESLint (flat config, `next/core-web-vitals` + `next/typescript`)

## Key Constraints

- **Static export only** — no server-side features (no API routes, no `getServerSideProps`, no middleware, no `next/image` optimization). All pages must be statically generateable.
- **`trailingSlash: true`** — all routes end with `/`. Links must respect this.
- **No test framework** — there are no tests. Don't add test infrastructure unless asked.

## Stack

- Next.js 16 + React 19 + TypeScript (strict)
- Tailwind CSS 3 with `darkMode: 'class'`
- Framer Motion for animations
- MDX articles via `next-mdx-remote` + `gray-matter`, syntax highlighting with `shiki`/`rehype-pretty-code`
- Path alias: `@/*` maps to project root

## Content

Articles live in `content/articles/*.mdx` with frontmatter: `title`, `description`, `date`, `tags`, `readTime`. The `lib/articles.ts` module reads them at build time.

RSS is generated by `scripts/generate-rss.mjs` (runs before `dev` and `build`). Feed output goes to `public/rss.xml` and `public/rss/` directory.

## Conventions

- Components in `components/`, pages in `app/` (App Router).
- Dark mode via `ThemeProvider` component — uses `class` strategy with `localStorage`. Theme toggle inline script in `layout.tsx` prevents flash.
- Font: Geist via `next/font/google`, applied as CSS variable `--font-geist`.
- Styling: Tailwind utility classes. No CSS modules, no styled-components.
- When adding new articles, add the `.mdx` file to `content/articles/` — the slug is the filename.
